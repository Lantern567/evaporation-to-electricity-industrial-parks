# Data Processing Plan

本计划描述从本地研究方案到可复现数据管线的骨架流程。

## 1. Preserve Raw Data

原始气象、排污许可、企业、园区、能源价格和技术成本数据按来源保存在 `data/raw/` 或 `data/external/`。原始文件不做手工覆盖，所有清洗和转换由脚本完成。

## 2. Build Evaporation Deficit Indicators

- 整理逐日蒸发皿蒸发量、气温、风速、湿度、辐射和降水数据。
- 计算基准期自然蒸发能力，并构造县域、城市、园区和企业尺度 `EDI`。
- 必要时用 ERA5 或 CMFD 补充空间插值，形成可与园区和企业点位匹配的面板数据。

## 3. Construct High-Salinity Exposure

- 将煤化工、火电脱硫废水、石化化工、盐化工、印染、电镀、制药、光伏材料等行业标记为高暴露组。
- 结合工业废水量、行业产量、排污许可水量、零排放项目案例和高盐废水系数，构造 `H_s`。
- 为低盐普通工业废水行业构造对照组，用于因果识别和安慰剂检验。

## 4. Estimate Causal Effects

基础识别模型：

```text
Y_{i,p,t} = beta * EDI_{p,t} * H_i + gamma X_{i,p,t}
          + alpha_i + delta_{p,t} + lambda_{s,t} + epsilon_{i,p,t}
```

其中 `Y` 包括污水处理电耗、综合能源消费、处理成本、碳排、蒸发池扩建投资和零排放设备运行强度。

稳健性检验包括极端低蒸发年份事件研究、滞后效应、替代蒸发指标、剔除极端干旱年份、使用风速或辐射异常作为气象驱动，以及对低盐行业做安慰剂检验。

## 5. Convert Evaporation Gaps Into Energy Gaps

工程放大模型以水盐平衡为核心：

```text
S_t = S_{t-1} + Q_t + P_t * A - E_t * A * phi(TDS) - D_t
```

当 `E_t` 下降时，若不扩池，则 `D_t` 需要由 MVR、低温蒸发、膜浓缩、蒸发结晶或外运处置承担；若扩池，则所需面积近似按 `1 / E_t` 增加。

## 6. Run Zero-Carbon Park Stress Tests

- 构建园区小时级源荷平衡：绿电出力、储能放电、外购绿电和传统备用需要覆盖原工业负荷与新增水处理电驱负荷。
- 定义设计失效：绿电匹配率低于目标、年碳强度超过阈值、峰值负荷超过容量、浓盐水池库容超过安全上限。
- 设置蒸发能力下降 5%、10%、15%，以及低风低光、电价上涨、高盐废水增长和园区扩建等情景。

## 7. Optimize Provincial Resource Allocation

建立两阶段随机混合整数规划：

- 第一阶段决定 2025-2030 年各省园区的设备和电力冗余投资。
- 第二阶段在不同蒸发、绿电、价格和废水量情景下调度设备运行。
- 比较传统方案 A（按历史平均蒸发量扩池）、传统方案 B（缺口全部上 MVR）和优化方案 C（水-电-热-储协同）。
